## Decisions made this session

- Use `@xterm/*` namespace packages (xterm 5.5+), not the old `xterm-addon-*` naming.
- Use `tauri::async_runtime::spawn` (not `tokio::spawn`) for the PTY event forwarder — Tauri v2 has its own runtime adapter.
- App-scoped hotkeys (not global) — avoids macOS Accessibility prompt on first launch. Decided at planning time; capabilities file reflects this.
- File tree hides dotfiles by default (`show_hidden: false` in `list_directory`). `Cmd+.` toggle deferred to v0.2.

- **30s-debounced mark_active**: ping from `TerminalPane.onData` activity.
- **ADR-0005** written: terminal-tabs-within-project-window (distinct from ADR-0004's project-level windows-not-tabs decision).

## Decisions made this session (summary)

- WebGL renderer for xterm with `allow-unsigned-executable-memory` entitlement (kept the entitlement for v0.1 — alternative was canvas fallback at 3× CPU cost).
- App-scoped accelerators (no global hotkeys yet) — avoids macOS Accessibility prompt.
- Per-project terminal persistence (memory cost is the user's responsibility; they can close tabs).
- File tree expansion does NOT persist across project switches (fresh each time; matches VS Code).
- `lastProject()` auto-restore on app launch (no asking).
