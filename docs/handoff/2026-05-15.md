## Decisions made this session (summary)

- WebGL renderer for xterm with `allow-unsigned-executable-memory` entitlement (kept the entitlement for v0.1 — alternative was canvas fallback at 3× CPU cost).
- App-scoped accelerators (no global hotkeys yet) — avoids macOS Accessibility prompt.
- Per-project terminal persistence (memory cost is the user's responsibility; they can close tabs).
- File tree expansion does NOT persist across project switches (fresh each time; matches VS Code).
- `lastProject()` auto-restore on app launch (no asking).

## What Was Tried and Abandoned
- **Approach**: Cmd+T fires in every window after Rust-side `emit_to(EventTarget::WebviewWindow { label })` scoping.
  - **Why we walked back**: Targeted emit alone isn't enough in Tauri v2. The frontend `listen()` from `@tauri-apps/api/event` is a wildcard (target=Any) and catches all emits regardless of target.
  - **Lesson**: For per-window menu events, also scope on the listener side via `getCurrentWebviewWindow().listen()`. Documented in the function comment in `src/lib/ipc.ts` so we don't repeat this.
- **Approach**: Bucket cycle hotkey ⌘⌥]/[ → ⌘I/⌘⇧I → ⌘J/⌘⇧J.
  - **Why we walked back**: ⌘⌥]/[ heavy to type repeatedly; ⌘I clashes with macOS italic in rich-text contexts.
  - **Lesson**: For high-frequency app-scoped hotkeys, pick single letters with no macOS system or webview default binding — ⌘J is a sweet spot ("Jump").
- **Approach**: Initial M2 terminal panel used `<Show keyed>` to remount on project switch.
  - **Why we walked back**: User then wanted opposite behavior (keep terminals alive across project switches).
  - **Lesson**: Per-project state should live in the parent component as a multi-project Map, not in the per-project subtree that gets remounted. TerminalsView now holds all tabs across all projects and filters by current projectPath.

## Decisions Made
- ADR-0001: Tauri v2 over Electron — ~5-10× RAM win without notarization complexity.
- ADR-0002: Solid over React — JSX-like + no VDOM + 6× smaller bundle.
- ADR-0003: rusqlite (bundled, WAL) over JSON — multi-writer safety from day one.
- ADR-0004: Separate windows over project-level tabs — macOS WindowServer handles Cmd-` for free.
- ADR-0005: Terminal-level tabs within a project window — distinct from ADR-0004 (different scope).
- ADR-0006: Window-project identity + navigate-vs-mutate split — `main` mutates, `project-<id>` is pinned, Recent/Cmd+P navigates.
- ADR-0007: Bucket model — ordered ring + per-bucket cursor + one app-active bucket; cycle reuses M3's spawn-or-focus.
- ADR-0008: Release process — ad-hoc-signed only for v0.1, notarization deferred until distribution demand exists.
