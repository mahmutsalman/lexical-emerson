# Handoff — 2026-05-16 (M7 shipped, ACL gotcha resolved)

This handoff supersedes the earlier `2026-05-16-workspace-bugs.md` (which was the
mid-session bug-report handoff). Keep that file for the diagnostic narrative;
this one is the close-out.

## Session Goal

Ship M7 — the Bucket Workspace, a cross-project terminal aggregator with a 2D
grouped tab-strip view and a 3D stacked-arcs immersive mode. The user wants to
have 3+ project windows each running their own `claude` plus build watchers,
and one workspace window that surveys all of them at once.

## What Got Done

- **PTY registry in `AppState`** — `Mutex<HashMap<pty_id, PtyTerminalInfo>>`
  populated atomically by `open_terminal` (same call that spawns the PTY also
  inserts into the registry, no two-step race). Drained by `PtyMessage::Exit`
  on PTY teardown.
- **New window family** `bucket-3d-<bucket_id>` spawned by
  `spawn_bucket_3d_workspace` Tauri command, routed in `App.tsx` by label
  prefix to mount `BucketWorkspace` instead of the regular launcher chrome.
- **`BucketWorkspace.tsx`** (~680 lines) — 2D grouped tab strip + 3D stacked-
  arcs view (extension of the per-project CSS-3D cylinder pattern), workspace-
  owned `+` terminals with `closeOnUnmount: false` so they outlive the
  workspace window, attached "live" tabs that mirror per-project PTYs.
- **3D-mode fixes** — `pointer-events: none` on inactive rings + `z-index: 5`
  on active ring (otherwise overlapping inactive-ring boxes capture clicks);
  `TerminalHandle.fitNow()` exposed from `TerminalPane` and called multiple
  times (30/120/260/480ms) after mode/active-project changes to work around
  WebKit's unreliable ResizeObserver-on-class-swap behaviour. Also `fitNow`
  on each newly-registered handle so panes mounting *during* 3D mode size
  correctly.
- **App.tsx hardened** — `<Show when={currentProject()}>` (replacing the
  earlier `when={projectPath()}` + `projectId() ?? 0` fallback) so the
  terminal panel never mounts with a placeholder `projectId=0` that would
  poison the registry.
- **ADR-0010** written: documents the PTY-registry pattern AND the
  load-bearing Tauri v2 capabilities-per-window-label trap (see below).

## The Bug That Cost the Most Time

Symptom: opening the workspace showed "0 tabs · registry 0" even though every
project window had a working terminal. We threw multiple incorrect fixes at
it (camelCase/snake_case rename, currentProject Show guard, fitNow retries)
and rebuilt three times — each install left the symptom unchanged.

Root cause (found on the fourth pass after enabling production devtools):
`src-tauri/capabilities/default.json` had `"windows": ["main", "project-*"]`.
The new `bucket-3d-*` window family was not listed, so it had **zero
permissions** at runtime. JS-side `listen("terminals://changed", ...)`
silently rejected with:

```
Unhandled Promise Rejection: Command plugin:event|listen not allowed by ACL
```

These rejections only appear in devtools — never in stderr, never in the UI.
So the workspace was running with a dead event listener: its initial
`await reconcile()` (using `invoke`, allowed by default) worked fine, but
every subsequent `terminals://changed` broadcast was lost. Same root cause
explained the dead ⌘T, ⌘⌥3, ring-navigation in the workspace — all use
`listen()` for menu events.

Fix: one-character JSON edit. `"windows": ["main", "project-*", "bucket-3d-*"]`.

## What Was Tried and Walked Back

- **Approach:** "Stale Vite bundle" theory — the previous build's
  source-file mtimes were minutes after the build start, suggesting Vite
  might have snapshotted before the latest edits.
  - **Why we walked back:** Rebuild produced identical symptom. Bundle was
    fine; the bug was deeper.
  - **Lesson:** When a clean rebuild doesn't move the needle, stop suspecting
    cache and start suspecting runtime config (permissions, ACLs, env).
- **Approach:** Tauri v2 camelCase→snake_case argument-name conversion.
  - **Why we walked back:** Confirmed via Tauri v2.11.1 source
    (`tauri-macros/src/command/wrapper.rs`: `argument_case: ArgumentCase::Camel`)
    that conversion is the default. Not the bug.
  - **Lesson:** When two independent web searches give contradictory
    accounts of framework defaults, go to the macro source.
- **Approach:** Two separate Tauri-app processes (per-window state isolation).
  - **Why we walked back:** `ps -axm | grep lexical-emerson` showed exactly
    one PID. Single process → shared state. Not the bug.
  - **Lesson:** `ps` ruled out an entire class of hypotheses in 5 seconds.
    Should have started there.
- **Approach:** Adding `#[tauri::command(rename_all = "camelCase")]`
  defensively to every command.
  - **Why we kept it:** No behavioural change (it's the default), but it
    makes the wire contract explicit. Belt-and-braces. Harmless.

## Decisions Made

- **ADR-0010** — the PTY registry pattern (atomic register on spawn, drain on
  Exit, single broadcast channel) AND the capabilities-per-window-label rule.
  Future window families MUST be added to `capabilities/default.json` or
  silently lose all event-based functionality.
- **Production devtools enabled** via the `devtools` feature on the tauri
  crate. Negligible size cost; surfaces this class of error immediately
  next time.
- **Diagnostic UI in workspace** is permanent (gated behind a header
  toggle, off by default). Debug strip surfaces the registry contents +
  Test-insert + Refresh. Future runtime bugs will be much faster to
  diagnose with this in place.
- **`eprintln!` traces** on every registry op are kept in Rust. Visible if
  the app is launched from a terminal; zero cost otherwise.

## Notes for Next Session

- The earlier `2026-05-16-workspace-bugs.md` handoff is now a historical
  artifact — it captured the broken state and the (mostly-wrong)
  hypotheses we worked through. Keep it for the diagnostic narrative; do
  not act on it.
- The PTY registry pattern from M7 is a good template for any future
  cross-window state aggregation. Three rules to remember:
  1. Atomic write inside the same command that creates the underlying
     resource (no two-step register).
  2. Drain on resource teardown (PtyMessage::Exit in our case).
  3. Broadcast a `*://changed` event so listeners reconcile.
- The ACL gotcha now has a permanent guard: production devtools surface
  the rejection in the console, and the debug strip in the workspace
  surfaces the empty-registry symptom. Future window-kind additions
  should still update `capabilities/default.json` *as part of the same
  commit*.

Two M7 polish items: make the 3D workspace's per-project notes face

## What Got Done
- Commit 68de62d — single commit bundling tonight's two fixes plus
  the in-progress notes-rail/ProjectNotesPanel work that had been
  sitting uncommitted (the shared `notes-quill.ts` module, the
  read-only `ProjectNotesPanel`, the per-project notes-rail CSS, and
  the per-project TerminalsView's notes-face slot).
- **Notes-face navigability (BucketWorkspace.tsx)**: introduced a
  `NOTES_SENTINEL = "__notes__"` value stored in `activeByProject`.
  `cycleTerminalInActive` now cycles `[notes, t0, t1, …]` in 3D mode
  (2D path unchanged). `ringRotationDeg` rotates the cylinder to slot
  0 when the sentinel is active. `reconcile` and the active-tab
  createEffect both preserve the sentinel and default empty projects
  (zero terminals) to it. The notes pane picks up an `is-facing`
  class when the sentinel is active.
- **CSS (bucket-workspace.css)**: added `.bw-pane.is-3d.is-notes.is-facing`
  with the focus drop-shadow, spelled out explicitly because
  `.is-notes` is later in source order and would otherwise win the
  cascade against the shared `.is-facing` rule.
- **Size-growing bug (BucketWorkspace.tsx)**: swapped the bw-rings
  transform from `translateY(...) rotateX(...)` to
  `rotateX(...) translateY(...)`. CSS applies the rightmost transform
  first to a point; with the old order, the active ring at local
  y = `activeIdx * ringHeightPx` got rotated around the bw-rings
  center BEFORE the translation, ending up at z =
  `activeIdx * ringHeightPx * sin(tilt)`. Positive on lower rings →
  pulled toward camera → perspective enlarged them. New order
  translates the active ring to local origin first, so rotateX(tilt)
  no longer shifts it in z. Inline comment in the source captures
  the math so we don't re-discover this.

- The user noted that during this session he did NOT restart the dev
  process — he learned that an existing 1420-port dev server can
  satisfy `npm run tauri dev` without launching a new one, but some
  bindings/triggers won't pick up unless the dev server is fully

  next session: ask whether a full restart has happened.
- The size-growing-with-tilt math is worth keeping in mind for any
  future 3D feature that mixes translateY with rotateX. CSS reads
  transforms left-to-right but applies rightmost first; intuition
  says the opposite, and that's how this bug got in.

  per-window Quill instances that share image-blot context via a

  the image blot will resolve images correctly.
- The notes face being "always opacity:1" (always visible as a peer
  of the active terminal) is intentional — that's how the user sees
  notes off to the side without having to navigate to them. The new
  `is-facing` class layers an additional drop-shadow on TOP of that
  baseline, it doesn't replace it.
