**Canvas freeze fix — WebGL retired** (`TerminalPane.tsx:148-169`): dropped the entire `WebglAddon` try/load block. Always loads `CanvasAddon` after `term.open(hostEl)`. Root cause: xterm's WebGL canvas, when nested inside ancestors with `transform-style: preserve-3d` and active 3D transforms, gets promoted to its own composited GPU layer; WebKit then doesn't re-composite the layer on subsequent draw calls — the buffer updates but the screen stays frozen until `fitAddon.fit()` reads `hostEl.offsetWidth` and forces reflow. That's why navigation (which schedules `fitNow()` at 30/120/260/480 ms) unfroze the canvas. WebGL's `onContextLoss` fallback was wired correctly but doesn't fire on this silent failure mode. Canvas renderer's drawing surface stays synced with its composited layer — bug eliminated at the source. ~100 kB drop in bundle size as a bonus.

**Dome cylinder arrangement** (`BucketWorkspace.tsx:795+, 1226-1247, 1300-1320`): new `domeLiftPx(angleDeg, n)` helper returns `ringHeightPx() * 0.18 * (|angle|/halfArc)`. Applied as a trailing `translateY(-lift)` in each slot's transform — both terminal slots and the notes face. CSS transform order: `rotateY(θ) translateZ(r) translateY(-lift)`; since rotateY preserves the world Y axis, the lift translates each slot vertically by the same amount regardless of its rotation. Facing slot stays at the ring's baseline; edge slots sit visibly up-and-out. `DOME_LIFT_FRAC = 0.18` is the single tunable constant.
